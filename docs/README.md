# Docs

- `product/`: especificacoes e documentos ativos do produto.
- `assets/`: logos e referencias visuais fora do runtime da aplicacao.
- `examples/`: arquivos de exemplo para importacao e fluxos de negocio.
- `marketing/`: materiais editoriais e textos de divulgacao.

Capturas locais, backups e clones de repositorios de referencia devem ficar fora de `docs/`.
